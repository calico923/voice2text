# 実装計画（Windows推論 + Mac入力）

## 目的
- Macのメモリ負荷を抑えつつ、リアルタイム音声入力を実現する。
- 音声入力はMac、推論はWindows（WSL2）に分離する。
- 「話す -> 文字になる -> 任意アプリへ貼り付け」を最短で実用化する。

## 採用方針（固定）
- 推論サーバ: `WSL2 + vLLM + mistralai/Voxtral-Mini-4B-Realtime-2602 (BF16)`
- 通信方式: `vLLM Realtime API` のイベント仕様に統一
- クライアント: Mac側は軽量CLI/アプリ（推論モデルは持たない）
- 量子化: BF16で性能・安定性に問題が出た場合のみ検証

## コミット運用
- `Parent`（`Txx`）が1つ完了するたびに、必ず `git commit` を作成する。
- 1つのコミットに複数 `Txx` を混在させない。
- コミットメッセージは `Txx: <要約>` 形式で統一する。
- `docs/parent-checklist.md` の該当 `Txx` 行更新をセットで実施する。
- フック導入: `bash scripts/install-git-hooks.sh`

- 完了条件:

- やること:
  - WSL2上で `vllm serve` 起動
  - ローカルで `/v1/realtime` 接続確認
- 完了条件:
  - ローカル接続で逐次文字起こしが確認できる
  - 起動手順書と疎通確認手順が再現可能

### 3. Windows検証クライアントで経路固定（目安: 0.5日）
- やること:
  - Windows本体CLIクライアントで `localhost` / `WSL IP` / `Windows host IP` を検証
  - 採用経路を1つに固定し、接続手順を確定
- 完了条件:
  - 採用経路で再現性ある接続が確認できる
  - Mac側実装前にネットワーク前提が固定されている

### 4. Mac CLIクライアント実装（目安: 1日）
- やること:
  - マイク入力を `PCM16 / 16kHz / mono` で取得
  - Realtimeイベントで送信し、戻りの認識結果を表示
- 完了条件:
  - Macから話した内容がリアルタイムでCLIに表示される

### 5. 貼り付け機能追加（目安: 0.5日）
- やること:
  - `final` のみ `pbcopy + osascript` で貼り付け
  - Auto Pasteトグル（デフォルトOFF）を実装
- 完了条件:
  - `partial` は表示のみ、`final` のみ貼り付けされる

### 6. 安定化（目安: 1日）
- やること:
  - 再接続処理と接続状態表示
  - `transcription_delay_ms` とチャンクサイズ調整
  - ログ保存ON/OFF
- 完了条件:
  - 30〜60分連続で運用し、重大停止なし

### 7. 評価と分岐（目安: 0.5日）
- やること:
  - 遅延、欠落率、VRAM推移、OOM有無を記録
  - BF16継続可否を判定
- 完了条件:
  - 継続（BF16）か量子化検証移行かの判断が明文化される

## 判定基準

### BF16継続
- OOMが発生しない
- 体感遅延が許容範囲
- 長時間運用で安定

### 量子化へ移行
- OOMが発生する
- 遅延が許容範囲を超える
- 長時間運用で不安定
